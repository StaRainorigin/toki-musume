# UI 重构计划

## 目标
让 UI 更好维护：统一设计 token、消除死代码、修复响应式 bug、理清数据流。

## 改动清单

### 1. 新增全局样式 `src/styles/variables.css`
- CSS 自定义属性：`--color-accent: #4a90d9`、`--color-border: #ddd`、`--radius-sm`、`--radius-md` 等
- 在 `main.ts` 中 import，所有组件的 `<style scoped>` 中直接用 `var(--color-accent)`

### 2. 新增 `src/composables/useController.ts`
- 将 `AppController` 实例化为单例，通过 `useController()` 获取
- 暴露响应式 state（`mode`、`activeGoal`、`slackCount`、`messages`）
- 消除 App.vue 中手动 callback 赋值 ref 的脆弱模式
- DebugPanel 不再接收 controller prop，直接 `useController()`

### 3. 修复响应式 bug
- **ChatPanel**：加 `watch(messages, () => nextTick(() => el.scrollToBottom()))` 自动滚动
- **StatusBar**：用 `useNow()` composable（每秒 tick 的 `Date.now()` ref）驱动已用时间显示
- **TodaySummary**：移除死的 `summary` prop、`workMinutes` 和 `goalProgress` 改为从 controller state 计算或移除

### 4. 清理死代码
- `StatusBar.vue`：删除未使用的 `.end-btn` CSS
- `App.vue`：删除 `workMinutes`、`goalProgress` 死 ref
- `controller.ts` 的 `emitState`：移除 `messages: []` 和 `workMinutes: 0` 死字段
- `App.vue` header 的 `settings-btn` class 改为 `icon-btn`

### 5. 统一消息类型
- `ChatPanel.vue` 和 `App.vue` 中的内联 `{ role; content; ts }` 改用 `controller.ts` 已导出的 `UIMessage` 类型

### 6. HistoryView 走 controller
- 在 `AppController` 加 `listHistory(startDate, endDate)` 方法
- HistoryView 不再直接 import `tauri-bridge`，通过 `useController()` 获取数据
- 加关闭按钮

### 7. SettingsPanel 完善
- 加取消/关闭按钮
- 暴露 `CompanionConfig.enabled` 和 `fallbackIntervalMinutes` 字段
- 提示用户也可以直接编辑 config.json 文件

### 8. DebugPanel 主题统一
- 改为浅色主题，与主界面一致
- 通过 `useController()` 获取数据，不再接收 prop

### 9. 组件文件结构整理
```
src/
  styles/
    variables.css        # CSS 自定义属性
  composables/
    useController.ts     # controller 单例 + 响应式 state
    useNow.ts            # 每秒 tick 的时间 ref
  components/
    ChatPanel.vue
    MessageInput.vue
    StatusBar.vue
    TodaySummary.vue
    SettingsPanel.vue
    HistoryView.vue
    DebugPanel.vue
  App.vue
  main.ts
```

## 不改动的部分
- Rust 后端不变
- controller.ts 核心逻辑不变（只改 `emitState` 和加 `listHistory` 方法）
- types.ts 不变
- 测试 mock 路径不变
