# Phase 4: Live2D 集成

## 概述
用 pixi-live2d-display 在 Tauri WebView 里渲染 Live2D 模型，根据角色状态（专注/休息/摸鱼/任务完成）切换表情和动作。

## 依赖安装
```bash
pnpm add pixi.js@7 @pixi/live2d-display
```
- pixi.js 7（pixi-live2d-display 的稳定版本依赖）
- @pixi/live2d-display（Live2D Cubism SDK Web 的 PIXI 封装）

## 模型文件
- 使用 Live2D 官方免费示例模型（Hiyori 或 Haru）
- 放到 `public/live2d/` 目录下
- 模型文件：`.model3.json` + textures + motions + expressions

## 新文件

### `src/components/Live2DCanvas.vue`
- PIXI Application 初始化
- 加载 Live2D 模型
- 鼠标跟踪（模型眼睛/头部跟随鼠标）
- 根据 `mode` + `pomodoroPhase` + `slackCount` 切换表情：
  - `rest` → 开心/放松表情
  - `focus` → 认真表情
  - 摸鱼检测触发 → 生气/不满表情（持续几秒后恢复）
  - 任务完成 → 庆祝表情
- 自动播放待机动作

### `src/composables/useLive2D.ts`
- 管理 Live2D 模型实例
- `setExpression(name)` — 切换表情
- `playMotion(group, index)` — 播放动作
- 暴露当前表情状态

## 修改文件

### `src/App.vue`
- 头像区域从 emoji Icon 替换为 Live2DCanvas 组件
- 或者作为浮动窗口/侧边栏显示

### `src/composables/useController.ts`
- 暴露 `slackCount` 和 `pomodoroPhase` 供 Live2D 读取状态

### `src/composables/useModeIcons.ts`
- 可以保留作为 fallback（当 Live2D 加载失败时用 Icon）

## 表情联动逻辑
| 状态 | 表情 | 触发 |
|------|------|------|
| 休息 | happy | mode === 'rest' |
| 专注中 | normal/serious | pomodoroPhase === 'focus' |
| 摸鱼检测 | angry | slack_detected 后持续 10 秒 |
| 专注结束 | surprised | pomodoro_focus_end |
| 任务完成 | happy/celebrate | task_completed |

## 不改动
- Rust 后端不变
- 摸鱼检测/番茄钟逻辑不变
- Live2D 只是 UI 层，不影响业务逻辑
