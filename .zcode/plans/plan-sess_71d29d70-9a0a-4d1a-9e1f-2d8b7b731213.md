# 番茄钟 + 今日任务系统设计

## 概述
在现有 Goal 系统上扩展，新增今日任务列表 + 番茄钟计时器。两种任务类型：计时任务（番茄钟）和目标任务（手动完成）。

## 数据结构

### Task 类型（新增 `src/types.ts`）
```ts
export type TaskType = 'timed' | 'target'

export type Task = {
  id: string
  title: string              // 任务标题
  type: TaskType             // timed=计时任务(番茄钟), target=目标任务
  mode: GoalMode             // study | work
  status: 'pending' | 'active' | 'completed' | 'abandoned'
  
  // 计时任务字段
  plannedMinutes?: number    // 计划总时长
  completedMinutes?: number  // 已完成时长
  pomodoroCount?: number     // 已完成番茄数
  
  // 目标任务字段
  description?: string       // 目标描述
  
  // 通用
  createdAt: number
  startedAt?: number
  completedAt?: number
}
```

### PomodoroState（新增 `src/state/pomodoro.ts`）
```ts
export type PomodoroPhase = 'focus' | 'break' | 'long_break' | 'idle'

export type PomodoroState = {
  phase: PomodoroPhase
  phaseStartedAt: number     // 当前阶段开始时间
  phaseDurationMin: number   // 当前阶段时长
  cycleCount: number         // 当前完成的 focus 次数
  // 配置
  focusMin: number           // 默认 25
  breakMin: number           // 默认 5
  longBreakMin: number       // 默认 15
  cyclesBeforeLongBreak: number  // 默认 4
}
```

## 新文件

### `src/state/task-store.ts` — 任务列表管理
- `TaskStore` 类：今日任务列表的增删改查
- `addTask(title, type, mode, plannedMinutes?)`
- `removeTask(id)`
- `completeTask(id)` — 标记完成
- `abandonTask(id)` — 放弃
- `getActiveTask()` — 当前正在做的任务
- `setActiveTask(id)` — 选一个任务开始
- `getTodayTasks()` — 今日所有任务
- 内存存储（和 AppProfileStore 一样，不依赖 Tauri）

### `src/state/pomodoro.ts` — 番茄钟状态机
- `createPomodoroState(config)` — 初始化
- `startFocus(state)` — 开始专注
- `checkPhaseEnd(state, now)` — 检查当前阶段是否结束
- `transitionTo(state, nextPhase)` — 阶段切换
- `getElapsedMin(state)` — 当前阶段已过分钟
- `getRemainingMin(state)` — 剩余分钟
- 专注结束 → 累加 `completedMinutes` 到当前 Task

### `src/llm/daily-plan-prompt.ts` — AI 今日计划
- `buildDailyPlanPrompt(history)` — 让 LLM 根据历史建议今日计划
- 返回 `Array<{ title, type, mode, plannedMinutes? }>`
- 用户确认后批量创建任务

## 番茄钟流程
1. 用户进入学习/工作模式 → 如果有 active task，启动番茄钟 focus
2. 25 分钟专注结束 → 累加 completedMinutes → 检查是否完成
3. 未完成 → 进入 5 分钟 break
4. break 结束 → 回到 focus
5. 每 4 个 focus → 15 分钟 long_break
6. completedMinutes >= plannedMinutes → 自动完成 task
7. 目标任务 → 用户手动点完成

## AI 今日计划流程
1. 早上打开应用 → controller 检查今天是否已有任务
2. 没有任务 → LLM 根据昨日总结 + 历史习惯建议 3-5 个任务
3. UI 显示建议列表 → 用户确认/修改/添加
4. 确认后批量创建任务
